**Fix slider.tsx (line 34):**
Remove `thumbAlignment="edge"` from the SliderPrimitive.Root props. This eliminates the script-tag injection that base-ui uses for edge alignment pre-hydration, which causes React strict mode console errors. The default thumbAlignment ('center') works correctly without the script injection path.

**Create EditRoomDialog.tsx:**
Create a dialog component that allows editing turnLimit and speakerStrategy for an existing room.

Props interface:
```typescript
interface EditRoomDialogProps {
  roomId: string;
  currentTurnLimit: number;
  currentSpeakerStrategy: 'round-robin' | 'llm-selected';
  disabled?: boolean; // true when room is running/paused
  onSaved?: (updated: { turnLimit: number; speakerStrategy: string }) => void;
}
```

Implementation details:
- Use Dialog, DialogTrigger, DialogContent, DialogHeader, DialogTitle, DialogDescription, DialogFooter from '@/components/ui/dialog'
- DialogTrigger renders a Button (variant="outline", size="sm") with text "Edit" — the button is disabled when `disabled` prop is true
- Local state: `turnLimit` (number, init from currentTurnLimit), `speakerStrategy` (string, init from currentSpeakerStrategy), `saving` (boolean), `error` (string | null)
- Reset local state to current values when dialog opens (use `onOpenChange` callback)
- Body contains: Slider for turnLimit (min=5, max=100, step=5) with numeric display, Select for speakerStrategy with Round-Robin and LLM-Selected options — same markup pattern as RoomWizard Step 1
- Slider onValueChange: use `Array.isArray(vals) ? vals[0] : vals` guard (same as RoomWizard)
- Select onValueChange: use `(val: string | null) => val && setSpeakerStrategy(...)` pattern (same as RoomWizard)
- DialogFooter contains Save button (disabled when saving) and Cancel (DialogClose)
- On Save: fetch PATCH `/api/rooms/${roomId}` with JSON body `{ turnLimit, speakerStrategy }`
  - On success (200): call `onSaved` with updated values, close dialog
  - On 409: set error to "Cannot edit while conversation is active. Stop the conversation first."
  - On other error: set error to "Failed to save changes"
  - Error displayed as `<p className="text-xs text-destructive">` above the footer
- Use `open` + `onOpenChange` controlled state on Dialog root so we can close programmatically on save success
