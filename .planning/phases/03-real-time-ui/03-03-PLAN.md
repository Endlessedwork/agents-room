<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 1: Verify complete real-time chat experience</name>
  <what-built>Complete real-time chat UI with SSE token streaming, agent identity display, thinking indicators, user message injection, and conversation controls (Start/Pause/Stop/Resume).</what-built>
  <how-to-verify>
1. Start the dev server: `npm run dev`
2. Open http://localhost:3000 in your browser
3. Navigate to a room that has 2+ agents assigned (create one if needed via the room wizard)
4. **Verify RTUI-01 — Live token streaming:**
   - Click "Start" in the chat header
   - Watch: agent messages should appear token-by-token (characters streaming in, not appearing all at once)
   - Confirm: no page refresh, no polling flicker — smooth real-time stream
5. **Verify RTUI-02 — Thinking indicator:**
   - When a new agent's turn begins, a bubble with animated bouncing dots should appear
   - The dots should use the agent's accent color
   - When the first token arrives, the dots should transition into streaming text (no flicker or layout jump)
   - When the turn completes, the blinking caret disappears
6. **Verify RTUI-03 — User message injection:**
   - While agents are conversing, type a message in the bottom input bar and press Enter
   - The message should appear right-aligned with a blue/primary bubble
   - The message should NOT interrupt the current agent's generation
   - On the next agent turn, check that the agent's response acknowledges or references your message (it entered their context)
7. **Verify RTUI-04 — Agent identity display:**
   - Each agent message should show: agent name (bold), role badge, model name (muted text)
   - Each agent should have a colored avatar circle on the left with their icon
   - The message bubble should have a colored left border matching the agent's color
   - Different agents should be visually distinguishable at a glance
8. **Verify controls:**
   - Click "Pause" — conversation pauses, header shows "Paused" badge
   - Click "Resume" — conversation continues from where it left off
   - Click "Stop" — conversation stops, header shows "Idle" badge, any thinking indicator clears
9. **Verify smart scroll:**
   - During streaming, chat should auto-scroll to keep latest content visible
   - Scroll up manually — auto-scroll should stop
   - A "scroll to bottom" button should appear
   - Click it or scroll back to bottom — auto-scroll should resume
10. **Verify system messages:**
    - If a system message appears (error, auto-pause), it should display as a full-width muted banner between message bubbles
  </how-to-verify>
  <resume-signal>Type "approved" if all checks pass, or describe specific issues that need fixing.</resume-signal>
</task>
