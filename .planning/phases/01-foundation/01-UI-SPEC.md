## Spacing Scale

Declared values (multiples of 4 only, 8-point base):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, status dot padding, inline badge spacing |
| sm | 8px | Compact row padding, button icon gap, form field gap |
| md | 16px | Default element spacing, card body padding, input horizontal padding |
| lg | 24px | Section padding, sidebar section header gap, wizard step padding |
| xl | 32px | Layout gaps between major sections, provider card grid gap |
| 2xl | 48px | Wizard page vertical padding |
| 3xl | 64px | Page-level vertical spacing (empty states, wizard top offset) |

**Exceptions:**
- Touch targets for icon-only buttons (sidebar toggle, delete action): minimum 44px × 44px per WCAG 2.5.5
- Temperature slider: 20px track height for usability
- Sidebar width: 240px fixed (not a spacing token — layout constant)
- Avatar size — small (in agent cards): 32px × 32px; large (agent form header): 48px × 48px

## Typography

Font family: **Sarabun** for all text. Declared in `src/app/layout.tsx` via `next/font/google`. Apply as `className={sarabun.variable}` on `<html>`, use `font-sans` throughout via Tailwind.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (Regular) | 1.5 | Room list items, form labels, provider card text, agent card body |
| Label | 12px | 600 (Semibold) | 1.4 | Status badges, section headings in sidebar, input field labels, wizard step labels |
| Heading | 20px | 600 (Semibold) | 1.2 | Page headings (Agent Library, Settings, Room name in main area), wizard step titles |
| Display | 28px | 600 (Semibold) | 1.2 | Wizard "Create Room" title, empty state headings |

**Weights declared:** 400 (Regular) and 600 (Semibold) only.

**Sarabun Google Fonts import:**
```typescript
// src/app/layout.tsx
import { Sarabun } from 'next/font/google';

const sarabun = Sarabun({
  subsets: ['latin', 'thai'],
  weight: ['400', '600'],
  variable: '--font-sans',
  display: 'swap',
});
```

## Color

Color tokens use shadcn CSS custom properties. Values resolve to light or dark variants based on `prefers-color-scheme`. Exact hex values are set at shadcn init time (Plan 01-03).

| Role | shadcn Token | 60/30/10 | Usage |
|------|-------------|----------|-------|
| Dominant (60%) | `--background` | 60% | Page background, main content area, wizard pages |
| Secondary (30%) | `--card` / `--muted` | 30% | Sidebar background, provider cards, agent cards, settings card panels |
| Accent (10%) | `--primary` | 10% | Reserved-for list below |
| Destructive | `--destructive` | semantic | Delete room button, remove agent action — no other uses |

**Accent (`--primary`) reserved for:**
1. Primary CTA buttons only ("Create Room", "Save Agent", "Add to Room", "Test Connection")
2. Active/selected room highlight in sidebar
3. Wizard step progress indicator (active step)
4. No other elements receive accent color

**Provider card status colors (locked — from CONTEXT.md):**

| Status | Color | Token | Element |
|--------|-------|-------|---------|
| verified | Green | `text-green-500` / `bg-green-500` | Status dot + "Connected" label |
| failed | Red | `text-red-500` / `bg-red-500` | Status dot + "Failed" label |
| configured (key saved, untested) | Yellow | `text-yellow-500` / `bg-yellow-500` | Status dot + "Not tested" label |
| unconfigured | Gray | `text-muted-foreground` / `bg-muted` | Status dot + "Not configured" label |

**Room status indicator colors (in sidebar):**

| Status | Color | Element |
|--------|-------|---------|
| running | Green dot | `bg-green-500` 8px circle |
| paused | Yellow dot | `bg-yellow-500` 8px circle |
| idle | Gray dot | `bg-muted-foreground` 8px circle |

## Component Specifications

### Sidebar

- Width: 240px fixed, full viewport height
- Background: `--card` (secondary surface)
- Contains: App name/logo at top, room list items, nav links at bottom (Agents, Settings)
- Room list items: name (14px/400) + topic truncated to 1 line (12px/400, `--muted-foreground`) + status dot (8px) + last activity timestamp (12px/400, `--muted-foreground`)
- Active room: `--primary/10` background tint, left border 2px `--primary`
- No agent avatars in the sidebar list (locked — CONTEXT.md)

### Provider Cards (Settings Page)

- Layout: 1-column stack (one card per provider)
- Card contains: Provider name + logo icon (Lucide or provider SVG) + status indicator + API key input field + "Test Connection" button
- Status indicator: 8px circle dot (color per status table above) + status label (12px/600)
- "Test Connection" button: secondary variant (not accent) — changes to loading spinner during test
- Ollama card only: additional "Custom host URL" input field below API key field
- 5 providers in order: Anthropic, OpenAI, Google, OpenRouter, Ollama

### Agent Cards (Agent Library)

- Layout: Responsive grid — 1 col on mobile, 2 col on tablet (≥768px), 3 col on desktop (≥1280px)
- Card contains: Avatar (32px circle — color + icon) + agent name (16px/600) + role excerpt (14px/400, 2 lines max) + provider badge + model name (12px/400) + temperature chip
- Avatar: colored circle with Lucide icon centered. Color from `avatarColor` (hex). Icon from `avatarIcon` (Lucide name).
- Preset template badge: shown only on agents created from presets — "Devil's Advocate", "Code Reviewer", "Researcher"
- Actions: "Add to Room" (in wizard context) or "Edit" + "Delete" (in library context)

### Agent Creation Form

Four structured fields — not a single textarea (locked — CONTEXT.md):

| Field | Label | Input Type | Placeholder |
|-------|-------|------------|-------------|
| promptRole | Role | Textarea, 3 rows | "e.g. You are a senior software architect with 15 years of experience..." |
| promptPersonality | Personality traits | Textarea, 3 rows | "e.g. Direct, skeptical, challenges assumptions before accepting them..." |
| promptRules | Rules | Textarea, 3 rows | "e.g. Always ask for evidence. Never agree without examining the premise..." |
| promptConstraints | Constraints | Textarea, 2 rows | "e.g. Do not use jargon. Keep responses under 200 words..." |

Additional fields: name (Input), avatar picker (color swatch grid + Lucide icon picker), provider Select, model Select (populated after provider selection), temperature Slider (0.0–1.0, step 0.1, default 0.7).

### Room Creation Wizard (3 steps)

Step 1 — Name your room:
- Input: Room name (required), max 60 chars
- Textarea: Topic / question (optional), max 280 chars, 4 rows

Step 2 — Pick agents:
- Agent library shown as compact list (not grid — wizard space constraint)
- Each row: avatar (24px) + name + provider badge + checkbox/add toggle
- Minimum 1 agent required to proceed
- Suggested agents highlighted (Claude's discretion — top 3 most recently created)

Step 3 — Review and create:
- Summary: room name, topic, selected agents (avatar + name list)
- Primary CTA: "Create Room" button
- Secondary: "Back" link

Step progress indicator: 3 numbered circles at top of wizard, active step uses `--primary`, completed steps use `--primary/60`, inactive steps use `--muted`.

### Empty Conversation Panel (Room view, Phase 1)

- Full main area, vertically centered
- No messages exist yet — conversation engine is Phase 2
- Shows room name (Display/28px) + topic (Body/14px) + agent avatars as a row of circles (32px each) + empty state copy

## Copywriting Contract

| Element | Copy | Context |
|---------|------|---------|
| Primary CTA — create room | "Create Room" | Wizard step 3 final button |
| Primary CTA — save agent | "Save Agent" | Agent creation form submit |
| Primary CTA — add to room | "Add to Room" | Agent library → room assignment |
| Primary CTA — test connection | "Test Connection" | Provider card action |
| Empty state heading — room list | "No rooms yet" | Sidebar when zero rooms exist |
| Empty state body — room list | "Create your first room to start a conversation between agents." | Below empty state heading in sidebar |
| Empty state heading — agent library | "No agents yet" | Agent library page with zero agents |
| Empty state body — agent library | "Create an agent with a persona, then assign it to a room." | Below empty state heading |
| Empty state heading — conversation panel | "Ready when you are" | Main area when room has no messages (Phase 1 state) |
| Empty state body — conversation panel | "Add agents to this room, then start a conversation in Phase 2." | Below empty state heading |
| Error state — provider test failed | "Connection failed. Check your API key and try again." | Provider card error state below button |

| Wizard step labels | "1. Name", "2. Agents", "3. Review" | Step progress indicator |
| Test connection loading | "Testing…" | Provider card button during test |
| Test connection success | "Connected" | Provider card status label after success |

## Interaction States

All interactive elements must implement these states:

| Element | Default | Hover | Active | Disabled | Focus |
|---------|---------|-------|--------|----------|-------|
| Button (primary) | `--primary` bg | 90% opacity | 80% opacity | 40% opacity, no pointer | 2px `--ring` offset ring |
| Button (secondary) | `--secondary` bg | `--accent` tint | pressed | 40% opacity | 2px ring |
| Button (destructive) | `--destructive` bg | 90% opacity | 80% opacity | 40% opacity | 2px ring |
| Input | `--border` outline | `--border` darker | — | `--muted` bg | 2px `--ring` |
| Sidebar room item | transparent | `--accent` bg | `--accent/80` bg | — | 2px inset ring |
| Provider card | `--card` bg | `--card` + shadow-sm | — | — | — |

Loading state: all async-triggering buttons replace label with a 16px spinning Lucide `Loader2` icon + text ("Testing…", "Saving…", "Creating…").
